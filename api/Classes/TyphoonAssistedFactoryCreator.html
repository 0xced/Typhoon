TyphoonAssistedFactoryCreator Class Reference

Inherits from NSObject
Declared in TyphoonAssistedFactoryCreator.h

Overview

Creates the full factory class from the given protocol description.

Users should not use this class directly.

Returns the selector of the only possible factory method of the protocol.

The protocol should only have one instance method, beside its property getters. Otherwise this method behaviour is undefined.

Other Methods

  • + creatorWithProtocol:returns:

    Returns a creator that will use the protocol as a template for the new class, trying to guess which method in the protocol corresponds to which init method in the returnType. If no match could be found, an error will happen during runtime when calling factoryClass.
  • + creatorWithProtocol:factoryBlock:

    Returns a creator that will use the protocol as a template for the new class. The protocol should only have one factory method. The factoryBlock will be used as implementation of such method. The arguments of the block are the arguments of the factory method in the same order, but the factory itself is prefixed as first argument, so the factory method arguments are the second and following arguments. If no factory method could be found, or several are found, an error will happen during runtime when calling factoryClass.
  • + creatorWithProtocol:factories:

    Returns a creator that will use the protocol as a template for the new class, while definitionBlock should contain one factory method definition for each of the factory methods in the protocol. If not enough factory method definitions, or more than necessary are found inside the block, an error will happen during runtime when calling factoryClass.
  • – factoryClass

    The Class of the factory created. If something was wrongly configured, this method will fail during runtime.
